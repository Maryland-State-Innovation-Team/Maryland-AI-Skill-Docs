# usa-in-page-navigation

USWDS sticky in-page navigation. A sidebar that lists the page's `<h2>` (and optionally `<h3>`) headings as anchor links, auto-generated by JS from the headings in the main content. The currently visible section is highlighted as the user scrolls, using the Intersection Observer API.

> **Different from `maryland-table-of-contents`.** The MDWDS TOC is an inline block at the top of the page; this is a **sticky sidebar** that stays visible as the user scrolls and auto-tracks the active section. Use this for long articles where users will scroll repeatedly through sections.

## What it looks like

A vertical sidebar to the left of the main content (typically `desktop:grid-col-3`). USWDS provides default styles, themed to Maryland's primary color via the CDN.

- The nav is **empty in the source HTML** — USWDS JS scans the main content for headings matching `data-heading-elements` (default `"h2, h3"`) and builds the link list on page load.
- Each generated link is a 14-16px sans-serif text link with a thin left border. The currently-visible section's link gets a thicker Maryland blue (`primary`) left border and bold/colored text.
- Links indent based on heading level: `<h2>` is the base level; `<h3>` is indented further right.
- **Sticky positioning** — at desktop, the nav uses `position: sticky` so it remains in view as the page scrolls. Mobile/tablet: it scrolls with the page.

The component requires both:

1. A `<aside class="usa-in-page-nav">` for the nav (empty — JS fills it).
2. A `<main>` (or any element) **immediately after** the aside containing the headings with `id` attributes.

The aside and main must be **siblings** inside a wrapping container.

## Variants

Configured via `data-*` attributes, not class modifiers:

| Attribute | Effect |
|---|---|
| `data-heading-elements="h2, h3"` | CSS selector for which headings to include. Default includes h2 and h3. Use `"h2"` for a flat single-level nav. |
| `data-threshold="0.2"` | Intersection-observer threshold (0–1). What fraction of a section must be visible to mark it active. |
| `data-scroll-offset="0"` | Pixel offset for active detection. Increase if a sticky header would otherwise cover the heading. |
| `style="position: relative"` | Inline override to disable sticky behavior. |

## Default markup

```html
<div class="usa-in-page-nav-container">
  <aside
    class="usa-in-page-nav"
    data-heading-elements="h2, h3"
    data-threshold="0.2"
    data-scroll-offset="0"
  >
    <nav aria-label="In-page navigation">
      <!-- USWDS JS auto-fills this with links based on headings in the next sibling -->
    </nav>
  </aside>

  <main id="main-content-article" class="usa-prose">
    <h2 id="introduction">Introduction</h2>
    <p>The Maryland Department of Natural Resources offers recreational fishing licenses online or at any DNR Service Center.</p>

    <h2 id="eligibility">Eligibility</h2>
    <p>Anyone 16 or older fishing in Maryland's tidal waters needs a license. Some exceptions apply for seniors and disabled veterans.</p>

    <h3 id="resident-licenses">Resident licenses</h3>
    <p>Maryland residents pay reduced rates and can buy 1-day, 7-day, or annual licenses.</p>

    <h3 id="non-resident-licenses">Non-resident licenses</h3>
    <p>Visitors from out of state can purchase the same license tiers at higher rates.</p>

    <h2 id="how-to-apply">How to apply</h2>
    <p>Apply online through the DNR licensing portal, by phone, or in person at a Service Center.</p>

    <h2 id="renewals">Renewals</h2>
    <p>Licenses expire each year on the last day of December and can be renewed starting November 1st.</p>
  </main>
</div>
```

JS generates approximately:

```html
<nav aria-label="In-page navigation">
  <ul class="usa-in-page-nav__list">
    <li class="usa-in-page-nav__item">
      <a href="#introduction" class="usa-in-page-nav__link">Introduction</a>
    </li>
    <li class="usa-in-page-nav__item">
      <a href="#eligibility" class="usa-in-page-nav__link usa-current">Eligibility</a>
      <ul>
        <li class="usa-in-page-nav__item usa-in-page-nav__item--sub-item">
          <a href="#resident-licenses" class="usa-in-page-nav__link">Resident licenses</a>
        </li>
        <li class="usa-in-page-nav__item usa-in-page-nav__item--sub-item">
          <a href="#non-resident-licenses" class="usa-in-page-nav__link">Non-resident licenses</a>
        </li>
      </ul>
    </li>
    <!-- ... -->
  </ul>
</nav>
```

## What each class does

| Class | Effect |
|---|---|
| `usa-in-page-nav-container` | Wraps the aside + main pair. Flex/grid layout at desktop. |
| `usa-in-page-nav` | The aside element. Sticky positioning at desktop (`position: sticky`); scrolls with page on mobile. Empty until JS populates. |
| `usa-in-page-nav__list` (auto-generated) | The `<ul>` inside the nav. No bullets, vertical stack. |
| `usa-in-page-nav__item` (auto-generated) | Each `<li>`. Adds a left border that becomes the active-state indicator. |
| `usa-in-page-nav__item--sub-item` (auto-generated) | A nested `<li>` for `<h3>`-level sub-items. Indented further right. |
| `usa-in-page-nav__link` (auto-generated) | The anchor. Text color `base-darker`, no underline. Hover: `primary`, underline. |
| `usa-current` (auto-applied) | Marker on the currently-visible section's link. Thicker `primary` left border, semibold, `primary` text. |

USWDS exposes additional spacing/scale tweaks via the upstream USWDS settings (`$theme-in-page-nav-...`), but you don't need to set those on a page.

## Prop schema

| Prop | Type | Default | Notes |
|---|---|---|---|
| `sticky` | bool | `true` | When `false`, the story sets `style="position: relative"` on the aside to disable sticky behavior. |
| `headingSelector` | string | `"h2, h3"` | CSS selector for headings to include in nav (passed as `data-heading-elements`). |
| `threshold` | number | `0.2` | Intersection-observer threshold 0–1. |
| `scrollOffset` | number | `0` | Pixel offset for active-section detection. Set to ~100 if you have a sticky header. |
| `contentSections` | array | sample 10-section list | Storybook-only — real pages emit their own headings in the `<main>`. |
| `enableAnalytics` | bool | `false` | Adds `data-ga-category`/`data-ga-action`/`data-ga-label` attributes for tracking link clicks. |
| `gaCategory` / `gaAction` / `gaLabel` | string | — | GA tracking values. |

## Heading level adjustment

The nav's own heading is provided by `<nav aria-label="In-page navigation">` (a landmark name, not a visible heading). Headings in the main content stay at their normal semantic levels — `<h2>` for section heads, `<h3>` for sub-sections. **All headings to be linked must have an `id` attribute** that matches the anchor href the JS will generate (slugified from the heading text by default).

## Common mistakes

1. **Putting the aside inside the main, or main inside the aside** — they must be **siblings** inside `usa-in-page-nav-container`. JS walks `nextElementSibling` to find the content; nested structures break detection.
2. **Forgetting `id` attributes on headings** — without them, the JS can't generate anchor links. Every `<h2>`/`<h3>` to be linked needs an `id`.
3. **Loading the USWDS JS twice** — the JS attaches a singleton on the aside. Re-initialization throws duplicate Intersection Observers and the active state flickers.
4. **Using this **and** `maryland-table-of-contents`** — pick one. The TOC is inline; this is a sticky sidebar. Both pointing to the same anchors is redundant.
5. **Setting `data-scroll-offset` to 0 when the page has a sticky header** — the section's top will be hidden behind the header. Set the offset to your sticky-header's height (~64px is typical).
6. **Using this on short pages** — if the page has only 2-3 sections, the sidebar takes up space without aiding navigation. Use this on pages with 6+ sections.
7. **Wrapping the empty nav in `usa-prose`** — JS-generated content is added to the empty aside. `usa-prose` would apply paragraph/list styling that fights the nav styles.

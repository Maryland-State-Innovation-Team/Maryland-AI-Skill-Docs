# maryland-table-of-contents

A boxed "On this page" navigation block for long-form pages. Sits at the top of an article (after the hero) and lists section anchor links, helping users jump to a specific section in the body.

> Not to be confused with `usa-in-page-navigation`, which is a sticky sidebar TOC auto-generated by JS from headings in the page. The `maryland-toc` is a **content-block** that appears inline at the top of the article. See `cdn/components/usa-in-page-navigation.md` if you want the sticky sidebar pattern.

## What it looks like

A wide horizontal block with a **3px Maryland-black (`ink`) top border** and a **1px `gray-cool-40` bottom border**. Right padding is 80px (`units(10)`) so the link list doesn't run flush to the right edge of its container. Bottom padding 40px (`units(5)`) and 32px (`units(4)`) margin-bottom separate it from the article body that follows.

**Heading** ("On this page") — 14px (`body-7`) with `units(3)` vertical margin, trimmed (no descender padding). Renders as an `<h2>` by default.

**Link list** — 16px (`body-6`) Maryland-styled bullets (USWDS list styles with secondary-color markers). Each link uses the MDWDS `typeset-link` mixin: Maryland blue (`primary`) color, underlined.

**Multi-column behavior:** at tablet (≥640px), if the list has 4 or more items it splits into 2 columns via CSS `column-count: 2`. Below tablet, it's a single column.

**Anchor scroll behavior:** any heading with an `id` attribute (the targets of the anchor links) gets a `scroll-margin-top: units(4) + var(--maryland-nav-inset-top, 0)` so scrolled-to sections sit below any sticky nav.

## Default markup

```html
<div class="maryland-toc">
  <h2 class="maryland-toc__title">On this page</h2>
  <ul class="maryland-toc__list">
    <li><a href="#eligibility">Who is eligible</a></li>
    <li><a href="#required-documents">Required documents</a></li>
    <li><a href="#fees">Fees and payment</a></li>
    <li><a href="#how-to-apply">How to apply</a></li>
    <li><a href="#renewals">Renewals</a></li>
  </ul>
</div>

<div class="usa-prose">
  <h2 id="eligibility">Who is eligible</h2>
  <p>Maryland residents 18 years or older may apply for a recreational fishing license.</p>

  <h2 id="required-documents">Required documents</h2>
  <p>Bring proof of Maryland residency and a valid government-issued ID.</p>

  <!-- ... more sections, each with an id matching the TOC -->
</div>
```

## What each class does

| Class | Effect |
|---|---|
| `maryland-toc` | The outer block. 3px solid `ink` top border, 1px `gray-cool-40` bottom border, no left/right borders. 80px right padding (`units(10)`), 40px bottom padding (`units(5)`), 32px bottom margin (`units(4)`). Applies USWDS list-styles to descendant `<ul>` elements. |
| `maryland-toc__title` | The "On this page" heading. 14px (`body-7`) with `units(5)` line-height. Trimmed top/bottom (no descender space). 24px (`units(3)`) vertical margin, zero left margin, 24px right margin. Default element: `<h2>`. |
| `maryland-toc__list` | The link list `<ul>`. 16px (`body-6`) base font. Splits into 2 columns at tablet+ if it has 4 or more items (CSS container `:has(li:nth-child(4))` selector). Uses the MDWDS USWDS-list-styles for marker color. |
| `maryland-toc a` | Each anchor link inside the TOC. Maryland blue (`primary`) text with an underline (via `typeset-link` mixin). |

Note: the TOC has no BEM modifier for the individual `<li>` or `<a>` — they're styled by descendant selectors on the container. Just add `<li><a href="#section">Section name</a></li>` for each item.

## Prop schema

| Prop | Type | Default | Notes |
|---|---|---|---|
| `sectionCount` | number | `5` | Used by the Storybook demo only — number of section links to generate. In production you write the `<li>` items by hand, one per `<h2 id="...">` in the article body. |

There are no runtime variants — the component is structural.

## Heading level adjustment

The TOC heading is an `<h2>` ("On this page"). This is correct when the TOC is at the top of `<main>` underneath the page's `<h1>` (the hero title). If you nest the TOC inside another `<h2>`-led section (rare), demote it to `<h3>`.

## Common mistakes

1. **Forgetting `id` attributes on the target headings** — the anchor links go nowhere if the corresponding `<h2 id="eligibility">` (etc.) doesn't exist. Make every TOC item's `href="#foo"` match an `id="foo"` in the body.
2. **Using `<a name="...">` instead of `id="..."`** — the modern pattern is `id` on the heading itself, not legacy named anchors.
3. **Mixing prose and the TOC inside one wrapper** — the TOC is its own block with its own borders. Don't wrap the TOC inside a `<div class="usa-prose">` (the prose styles will override the heading typography).
4. **Adding `usa-list--unstyled`** — the MDWDS TOC explicitly wants bullets (via the secondary-color markers from `usa-list-styles`). Don't strip them.
5. **Skipping the heading** — the `<h2 class="maryland-toc__title">` is required for screen-reader users to understand the block is a navigation aid. Don't render the bare list.
6. **Using this for sticky in-page navigation** — `maryland-toc` is an inline block at the top of the page. If you want a sidebar that scrolls with the user and tracks the active section, use `usa-in-page-navigation` instead.
7. **Adding a `<nav>` wrapper** — the TOC is a `<div>` by convention (the heading + list are sufficient for AT users). Adding `<nav>` introduces an extra landmark that doesn't add value.
